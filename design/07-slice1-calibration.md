# Slice 1 — Calibration corpus + benchmark (design)

**Goal:** convert prism from *cited* to *measured*. Build a labeled corpus, run prism's lenses over it, and
publish — on prism's **own** data — the actual pairwise-agreement matrix, per-lens precision/recall, the
submodular-coverage gain, and a results table against CodeJudgeBench. Ship an `eval/` pack + a `prism eval`
command. This unlocks the three `design/01` deferrals (chance-corrected agreement, AST-edit-distance ρ, the
L5 lens — gated on data).

This doc is the study-swarm output (Research grounding, below) + the resulting design. It is a **plan**, not
shipped code; build begins only after the Phase-6 review.

## Research grounding

### A. Corpus construction (don't ship a gameable corpus)

- **Real faults over seeded ones.** Defects4J (Just et al., ISSTA 2014, doi:10.1145/2610384.2628055) **[V]** and
  BugsInPy (Widyasari et al., ESEC/FSE 2020, doi:10.1145/3368089.3417943) **[V]** give isolated, test-reproducible
  real bugs with minimized fixes. → **CODE positives anchor in real bug-fix commits**, Python-first via BugsInPy.
- **Mutants are a *minority augment*, not the backbone.** Just et al. (FSE 2014, doi:10.1145/2635868.2635929) **[V]**:
  mutant detection correlates with real-fault detection, but **~17% of real faults couple to no common mutant**. →
  cap synthetic mutants at a minority of CODE positives; bias operators to the ones that couple best.
- **Mirror the real fault distribution.** ManySStuBs4J (Karampatsis & Sutton, MSR 2020, doi:10.1145/3379597.3387491)
  **[V]**: 153k single-statement fixes, 16 templates, ~1 bug / 1.6–2.5k LOC. → sample planted-bug *types* in
  template proportion; clean ≫ buggy in reality.
- **Contamination is real and inflates scores.** "Data Contamination Through the Lens of Time" (Roberts et al.,
  2023, arXiv:2310.10628) **[V]**: LLM pass-rates trend with problem *release date* + GitHub popularity. → build a
  **fresh / held-out split** (commits dated after verifier-model cutoffs + author-written novel bugs), reported
  **separately** from the public split, with a date-stratified **contamination-delta** column.
- **Class balance distorts P/R.** Saito & Rehmsmeier (PLOS ONE 2015, doi:10.1371/journal.pone.0118432) **[V]**:
  ROC is deceptive under imbalance; PR is honest. → sample **balanced for recall stability** (≥100 positives/lens
  → ±~8% Wilson interval) but **also report precision re-weighted to realistic prevalence**.
- **The verifier must be independent of the generator.** Huang et al. (ICLR 2024, arXiv:2310.01798) **[V]** +
  Kambhampati et al. (ICML 2024, arXiv:2402.01817) **[V]**: self-correction fails; verification needs an external
  checker. → label which samples were generated by prism's own verifier family; measure with the generator's
  reasoning hidden, so the benchmark proves prism isn't grading its own homework.

### B. Metrics (what the eval report computes)

- **Chance-corrected agreement, not raw overlap.** Cohen's κ (1960) **[C]**; Hayes & Krippendorff (2007,
  Communication Methods and Measures) **[C]**. Krippendorff's **α** handles ≥3 raters + abstentions natively. →
  **headline diversity = Krippendorff's α**, beside the **pairwise Cohen's-κ matrix** to localize collapse.
- **Guard the kappa paradox.** Feinstein & Cicchetti (1990) **[C]**: under skewed prevalence, high agreement →
  near-zero κ. prism's corpus is prevalence-skewed (defects are rare). → report κ **with** raw agreement +
  per-lens prevalence + **PABAK**; never read a low κ as "healthy diversity" without checking prevalence.
- **Jaccard ≠ agreement.** It ignores jointly-absent items (two lenses correctly flagging *nothing* on a clean
  sample). → keep Jaccard only as a *finding-co-location* view, not the decision-agreement metric.
- **Structural similarity for code findings.** Song et al. (ACL 2024, arXiv:2404.08817) **[V]**: AST/tree-edit
  distance (TSED) beats BLEU/token overlap for code. → code-finding ρ uses tree-edit distance (Zhang–Shasha/APTED),
  so two lenses proposing the same fix with different names/whitespace read as *agreeing*.
- **MCC over F1/accuracy under imbalance.** Chicco & Jurman (BMC Genomics 2020, doi:10.1186/s12864-019-6413-7)
  **[C]**. → per-lens quality headlines **MCC + balanced accuracy** + full confusion matrix; accuracy alone is
  banned (a rubber-stamp lens scores ~95% acc, MCC≈0 — the exact failure prism exposes). PR-AUC if a lens emits a
  continuous score.
- **Calibration is a required separate axis.** Guo et al. (ICML 2017, arXiv:1706.04599) **[C]**: modern models are
  over-confident. prism *returns a confidence*. → report **ECE + Brier + reliability diagram**; temperature-scale
  on a held-out split if over-confident.

### C. The ρ ≤ 0.25 threshold — the roadmap's #1 gap

- **0.25 is descriptive, not normative.** Rajan (2025, arXiv:2511.16708) **[V]**: ρ = 0.05–0.25 is the *observed*
  correlation **ceiling among already-diverse agents** on 99 samples — it never proposes "refuse if ρ > 0.25", and
  doesn't pin the ρ metric (Pearson/Spearman/MI). **prism has been repurposing a measured upper bound as a
  refusal gate.** → **stop asserting 0.25**; calibrate prism's own cutoff on labeled data.
- **Diversity beats count, with steep diminishing returns.** Rajan **[V]**: multi-agent +39.7pp (32.8→72.4%), with
  per-agent marginal gains +14.9/+13.5/+11.2pp. Correlated errors are higher within a model family — Kim et al.
  (ICML 2025, arXiv:2506.07962) **[V]** — which is *why* Lock 1 (family-different) makes a low ρ achievable. →
  the four-lens default is supported; report the **per-lens marginal coverage-gain curve** stratified by
  verifier-family pairing.
- **Don't trust ρ alone.** Kuncheva & Whitaker (Machine Learning 2003) **[C]**: ten diversity measures relate to
  ensemble accuracy only **weakly and inconsistently**. Wood/Brown et al. (JMLR 2023) **[C]**: there is a formal
  bias–variance–**diversity** decomposition (diversity isn't free). → co-report a **direct submodular
  coverage-gain** metric (Nemhauser–Wolsey–Fisher 1978 **[C]**): `f(all) − max_i f({lens_i})` + the marginal-gain
  curve; a lens whose marginal gain ≈ 0 is the operational definition of "redundant," a *stronger* collapse signal
  than ρ. ρ becomes a cheap secondary tripwire.
- **Calibrate by sweep, report an interval.** Youden J (1950) **[C]** to pick the cutoff maximizing
  collapse-detection; Cawley & Talbot (JMLR 2010) **[C]**: a threshold tuned + scored on the same small corpus is
  optimistically biased → **nested CV**, report the cutoff as a **CV'd interval with CI**, not a point. If the CI
  spans (say) [0.18, 0.34], that *is* the honest finding — and it argues for keeping 0.25 as a documented default
  with the existing per-pair override, not a precision claim.

### D. Benchmark validity (so prism's own numbers are trustworthy)

- **Demonstrate the family-different lock.** Panickssery et al. (NeurIPS 2024, arXiv:2404.13076) **[V]**: LLMs favor
  their own generations, *linearly* in self-recognition. → headline experiment = **same-family vs different-family
  A/B** on identical artifacts; publish the accuracy delta vs human gold as the reason there is no same-family
  fallback.
- **Position/order bias can invert a verdict.** Wang et al. (ACL 2024 Findings, arXiv:2305.17926) **[V]**; Shi et
  al. (arXiv:2406.07791) **[V]**. → score every pairwise item in **both orders**, count correct only if
  order-consistent, report a **position-consistency rate**; use per-example position-entropy as the
  escalate/uncertainty gate (ties to UNCERTAINTY_GATED_HUMANS).
- **Cap claims at the human ceiling.** Zheng et al. (NeurIPS 2023, arXiv:2306.05685) **[V]**: GPT-4 ↔ human
  agreement ≈ **human ↔ human (~80%)** on subjective axes. → don't market verifier accuracy above the inter-human
  bound without flagging it; prefer **executable ground truth** (tests pass/fail) where possible.
- **Report against CodeJudgeBench honestly.** Jiang et al. (2025, arXiv:2507.10535) **[V]**: pairwise judging across
  codegen/repair/testgen; significant run-to-run randomness; order changes accuracy; **pairwise > pointwise**. →
  report **per-task + per-producer-split**, pairwise framing. **Disclosed tension:** the benchmark finds retaining
  reasoning *helps* the judge, but prism **strips reasoning by contract (Lock 2)** — report against the *stripped*
  condition and state the deliberate **accuracy-for-integrity trade**.
- **Robustness is table-stakes.** CodeJudgeBench-Adversarial (2026) **[V]**: cosmetic perturbations collapse single
  judges (some worse than random). → publish a **clean-vs-adversarial** accuracy pair — also the strongest external
  evidence for the multi-lens lock.
- **Non-determinism → report variance.** → every benchmark cell = **N≥3 runs at a pinned temperature**, mean ± CI,
  with model/seed/prompt-hash in the receipt (PIN_PER_STEP). A figure with no variance band doesn't ship.
- **Bias-sensitivity is a first-class column** (Zhao et al. 2026, arXiv:2604.16790 **[V]**); build gold from
  executable truth or a **cross-family jury** (PoLL — Verga et al. 2024, arXiv:2404.18796 **[V]**), never one model.

### E. The L5 Style/Maintainability lens — gate on data

- Bacchelli & Bird (ICSE 2013) **[V, full-text]**: defects are only ~14% of review comments; the bulk is
  maintainability/readability — the *subjective* part. Reviewer notions of quality diverge (Kononenko et al.,
  ICSE 2016, doi:10.1145/2884781.2884840 **[C]**). Maintainability metrics are weak proxies: MI is an unvalidated
  regression fit (Coleman et al. 1994 **[C]**), CC is ~redundant with SLOC (Landman et al., ICSME 2014 **[C]**),
  smells barely predict effort (Sjøberg et al., TSE 2013 **[C]**), and even Cognitive Complexity's author concedes
  it doesn't measure maintainability (Campbell 2023 **[V, full-text]**). LLM judges match humans only on *large*
  quality gaps (Zheng 2023 **[V]**) and rarely reproduce human review comments (Pornprasit & Tantithamthavorn 2024,
  arXiv:2402.00905 **[V]**).
- **Decision:** Slice 1 does **not** ship L5; it builds the **gate that decides L5**. If shipped later, L5 is
  **narrow** (locally-decidable style defects only: dead code, unused symbols, intra-diff naming/format
  inconsistency — *no* scalar maintainability scores, *no* MI, *no* whole-design judgments), and clears a bar:
  ≥100 items, ≥2–3 independent human labels, **Krippendorff α ≥ 0.6**, and **L5 precision ≥ 0.8** per category.
  **Submodularity guard:** L5 alone may never trigger ESCALATE (corroborate/REVISE-annotate only); drop any L5
  category whose findings correlate with SLOC or with the correctness lenses. Default if the corpus doesn't clear
  it: **stay deferred**.

## The design

### 1. Corpus (`eval/corpus/`)
Three artifact classes, each with a `public/` and a `fresh/` (held-out, post-cutoff + author-authored) split:
- **code/** — buggy vs clean. Real bugs (BugsInPy-derived, attribution preserved) + a minority of template-sampled
  planted bugs (ManySStuBs4J distribution). Each sample: `{id, artifact, intent, label, bug_class, source,
  split}`.
- **citations/** — real vs fabricated/corrupted/numeric-mismatch (reuse the `test_meta_citations` trap generators).
- **tool_calls/** — boundary-leak vs clean (BoundaryLens contract classes).
- Balanced within each lens's positive class (≥100 positives/lens); a `prevalence.json` records realistic priors
  for precision re-weighting. A `MANIFEST.json` pins counts, splits, sources, and a contamination note.

### 2. Metrics module (`eval/metrics.py`)
Pure, deterministic functions: Krippendorff α + pairwise Cohen κ + PABAK; per-lens MCC, balanced accuracy,
precision/recall/F1, confusion matrix; PR-AUC (continuous scores); ECE + Brier + reliability bins; AST/tree-edit ρ
for code findings; submodular coverage gain (`f(all) − max single` + marginal-gain curve). Unit-tested with known
fixtures (these are the kappa/Krippendorff/AST-ρ deferrals, now implemented).

### 3. ρ-threshold calibration (`eval/calibrate.py`)
Treat max-pairwise-ρ per verification as a score, human "sound vs collapsed" as the label; sweep ρ ∈ [0, 0.5],
ROC + Youden J under **nested CV**, emit the cutoff **as an interval + CI**, stratified by verifier-family pairing.
Output recommends keep-0.25-as-default vs recalibrate, co-reported with the coverage-gain signal.

### 4. Benchmark runner (`eval/benchmark.py`)
Runs all lenses over the corpus (N≥3, pinned temp, both orders) → emits a report: diversity matrix, per-lens
quality, calibration, ρ-sweep, coverage gain, the **same-vs-different-family A/B**, clean-vs-adversarial, the
**bias-sensitivity block**, and a CodeJudgeBench results table (per-task/per-producer-split, stripped condition,
integrity-trade note). Provider calls are cassette-able for deterministic replay (the providers are mockable today).

### 5. `prism eval` command (`cli/main.py` → `eval/`)
`prism eval [--split public|fresh|all] [--lenses ...] [--report md|json] [--out eval/report/]`. Reads the corpus,
runs the runner, writes the report + a signed run-receipt (PIN_PER_STEP), prints a summary table. `--offline`
uses cassettes.

### 6. Outputs
`eval/report/` (git-ignored run output) + a committed `eval/RESULTS.md` snapshot the README/landing scorecard cites
**from actual numbers** (no estimates). The L5 gate verdict lands here.

## Build plan (proposed waves, for Phase-6 review)
1. **Corpus + metrics** (the foundation; metrics unit-tested against fixtures).
2. **Runner + `prism eval`** (wire lenses → report; cassette replay).
3. **Calibration + benchmark** (ρ sweep, family A/B, CodeJudgeBench table, adversarial + bias-sensitivity).
4. **L5 gate evaluation** → ship-or-defer verdict in `RESULTS.md`.
Each wave: build + `ruff`/`mypy --strict`/`pytest` green; domain agents with exclusive ownership (new domain:
`eval`).

## Standards compliance (the six)

| Standard | Score | Evidence / plan |
|---|---|---|
| PIN_PER_STEP | 2→3 | The benchmark pins model/seed/temp/prompt-hash per run into a signed run-receipt; corpus `MANIFEST.json` pins sample provenance + splits. Target 3 once the run-receipt is implemented + tested. |
| ANDON_AUTHORITY | 2 | `prism eval` halts on a corpus-integrity failure (missing labels, split leakage check) before reporting; a contamination-delta over threshold flags the run as untrustworthy rather than publishing a clean number. |
| NAMED_COMPENSATORS | 3 | See table below — eval is read-mostly; the one published artifact (`RESULTS.md` / scorecard) has a named revert. |
| DECOMPOSE_BY_SECRETS | 3 | New `eval/` module hides the corpus + metrics + calibration; `corpus/` (data) separate from `metrics.py` (pure math) separate from `benchmark.py` (provider orchestration). |
| UNCERTAINTY_GATED_HUMANS | 2 | The L5 gate + the ρ-calibration interval explicitly defer to a human decision when evidence is below bar (α<0.6, wide CI) — gated on uncertainty, not step count. Documented skip remains for prism-the-library's runtime path. |
| EXTERNAL_VERIFIER | 3 | The corpus labels are external ground truth; the same-vs-different-family A/B *is* the external-verifier experiment; CodeJudgeBench is a third-party benchmark. The whole slice exists to externally measure what prism asserts. |

A score < 2 requires remediation; none here. The two 2→ items have named targets (run-receipt impl; ANDON
integrity-gate test) inside the build waves.

## Compensators (NO-SKIP — irreversible actions in Slice 1)

| Irreversible action | Compensator | Surface | Owner | Post-rollback state |
|---|---|---|---|---|
| Commit a published benchmark number to `eval/RESULTS.md` / README scorecard | Revert the doc commit; regenerate from a re-run | `git revert <sha>` + `prism eval` | release operator | Prior (or no) published number restored |
| Corpus includes a contaminated/leaked sample that inflates results | Move sample to `public/` (never `fresh/`), recompute, annotate the delta | edit `MANIFEST.json` + re-run | eval owner | Honest split restored; delta disclosed |
| Provider calls during a benchmark run (paid compute) | Bounded by N runs + cassette replay; no world-state write to undo | `--offline` cassettes | eval owner | Read-only external; nothing to undo (documented) |

`prism eval` writes only the report dir + receipts (already covered by `receipt delete`/`prune`). It performs no
network writes and no irreversible world-touching action beyond the published-number commit above.
