# 

एजेंट वर्कफ़्लो के लिए रनटाइम पर निर्णय लेने वाली सेवा। परिवार-विभिन्न, तर्क-मुक्त, बहु-लेंस सत्यापन, जिसमें दोबारा देखे जा सकने वाले रसीद शामिल हैं। **[लैंडिंग पेज और हैंडबुक →](https://mcp-tool-shop-org.github.io/prism-verify/)**

## स्थापित करें

अपने PATH पर `प्रिज़्म` CLI (और HTTP सेवा) स्थापित करें:

```bash
uv tool install prism-verify        # or: pipx install prism-verify
```

क्या आपके पास पायथन नहीं है? npm लॉन्चर का उपयोग करें (डाउनलोड + SHA256-सत्यापित पूर्व-निर्मित बाइनरी):

```bash
npx @mcptoolshop/prism-verify verify --artifact @file.py --intent "..." --caller-family openai
```

या इसे एक लाइब्रेरी के रूप में जोड़ें - अतिरिक्त: `[एंथ्रोपिक]` `[ओपनएआई]` `[गूगल]` `[एमसीपी]` `[एचटीटीपी]` `[सभी]`:

```bash
uv add prism-verify
# or
pip install "prism-verify[all]"
```

## त्वरित शुरुआत

प्रिज्म हमेशा एक ऐसे मॉडल परिवार के साथ सत्यापन करता है जो कॉलर के परिवार से **अलग** होता है (लॉक 1), इसलिए कम से कम एक वैकल्पिक-परिवार प्रदाता को कॉन्फ़िगर करें। एक Ed25519 हस्ताक्षर कुंजी उत्पन्न करें (डिफ़ॉल्ट — सार्वजनिक कुंजी वाले किसी भी व्यक्ति द्वारा रसीद सत्यापित की जा सकती है) ताकि रसीद लिखी जा सके, या स्थानीय परीक्षण के लिए `PRISM_DEV=1` का उपयोग करें:

```bash
prism keygen --out ~/.prism/signing_key.pem      # Ed25519 keypair (default signing)
export PRISM_SIGNING_KEY=~/.prism/signing_key.pem # or: export PRISM_DEV=1 (local play)
export ANTHROPIC_API_KEY="sk-ant-..."             # alt-family verifier for an OpenAI-family caller

prism verify \
  --artifact @myfile.py \
  --intent "Sort a list in O(n log n)" \
  --caller-family openai \
  --provider anthropic
```

> पुरानी वैकल्पिक विधि: `export PRISM_SIGNING_SECRET="$(openssl rand -hex 32)"` HMAC का उपयोग करके रसीद पर हस्ताक्षर करता है (केवल उस साझा गुप्त कुंजी के धारकों द्वारा सत्यापित किया जा सकता है — [रसीद](#receipts--signing-ed25519-verifiable-by-anyone) देखें)।

## आर्किटेक्चर

प्रिज़्म API अनुबंध पर चार वास्तुशिल्प लॉक लागू करता है:

1. **परिवार-भिन्न** - कॉलर का मॉडल परिवार हमेशा सत्यापन से बाहर रखा जाता है।
2. **तर्क-मुक्त** - परिवार की सीमा पार करने से पहले निर्माता CoT को हटा दिया जाता है।
3. **बहु-लेंस** - कम से कम 3 स्वतंत्र लेंस समानांतर में चलते हैं।
4. **सबमॉड्यूलरिटी-जागरूक** - यदि लेंस बहुत अधिक सहमत हैं (संक्षिप्त संकेत) तो अस्वीकार कर देता है।

**उद्धरण** संबंधी कलाकृतियों के लिए, एक परतदार तल एलएलएम की प्रामाणिकता जांच से पहले चलता है — प्रत्येक निश्चित चरण उस चीज़ को अस्वीकार करता है जिसे वह *सिद्ध* कर सकता है, अन्यथा वह इससे दूर रहता है:

- **अस्तित्व तल** — लाइव arXiv/क्रॉसरेफ़ पुनर्प्राप्ति; एक निर्मित पहचानकर्ता को छोड़ दिया जाता है, इस पर तर्क नहीं किया जाता।
- **संख्यात्मक/इकाई तल** — एक प्रतिशत परिवर्तन, एक इकाई-स्केल विचलन (42 मिली- बनाम माइक्रो-आर्कसेक), या एक तुलना-दिशा संबंधी असत्यता (5.0 < 5.8 ≠ "अधिक") को अंकगणितीय रूप से पकड़ा जाता है।
- **प्रामाणिकता जांच** — पुनर्प्राप्त सार के विरुद्ध परिवार-भिन्न, तर्क-रहित एलएलएम की जांच।
- **लंबवत एनएलआई तल** *(वैकल्पिक, `PRISM_NLI_FLOOR`)* — एक एनकोडर एनएलआई क्रॉस-एनकोडर एक "समर्थित" कथन को अस्वीकार करता है जो एलएलएम ने दिया था, लेकिन एक यांत्रिक रूप से भिन्न मॉडल इसकी पुष्टि नहीं करता।

## कैलिब्रेशन और बेंचमार्क (`prism eval`)

प्रिज्म को इस तरह बनाया गया है कि इसे **मापा** जा सके, न कि केवल दावा किया जा सके। `prism eval` लेबल किए गए डेटासेट पर लेंस चलाता है और रिपोर्ट करता है — प्रिज्म के अपने डेटा पर — प्रति-लेंस परिशुद्धता/पुनर्प्राप्ति/एमसीसी, अंतर-लेंस विविधता मैट्रिक्स (क्रिपेंडॉर्फ़ α + जोड़ीवार कोहेन κ), उपमॉड्यूलर कवरेज-लाभ, निर्णय सटीकता और आत्मविश्वास कैलिब्रेशन (ईसीई/ब्रियर), प्रत्येक के लिए एक ईमानदार आत्मविश्वास अंतराल के साथ।

```bash
prism eval --split public --runs 3     # measure against the bundled corpus (needs a verifier)
prism eval --offline                    # deterministic mock (CI smoke; NOT a real measurement)
```

v0.5 रन (स्थानीय `mistral-small:24b`) ने एक मुख्य लॉक में एक वास्तविक अंतर उजागर किया: रनटाइम उपमॉड्यूलर मीट्रिक (खोज-सेट जैकार्ड ρ) प्रत्येक लेंस जोड़ी के लिए **0.0** पढ़ता है, जबकि निर्णय-स्तर कोहेन κ **0.73–0.81** है — `ρ ≤ 0.25` गेट *लेंस सहसंबंध κ को अनदेखा करता है*। यह पता लगाना स्लाइस का पूरा उद्देश्य है; पूर्ण परिणाम + विधि [`eval/RESULTS.md`](eval/RESULTS.md) और [`design/07`](design/07-slice1-calibration.md) में।

## HTTP सेवा

प्रिज़्म को एक HTTP सेवा के रूप में चलाएं (इसके लिए `[एचटीटीपी]` अतिरिक्त की आवश्यकता है):

```bash
prism serve --host 127.0.0.1 --port 8000      # OpenAPI docs at /docs
```

| एंडपॉइंट | यह क्या करता है |
|---|---|
| `POST /verify` | एक कलाकृति को सत्यापित करें (CLI के समान अनुबंध)। बजट के भीतर ब्लॉक; `पसंदीदा: एसिंक्रोनस-प्रतिक्रिया` + एक `वेबहुक` URL → `202`, निर्णय (हस्ताक्षरित) वेबहुक पर भेजा जाता है। |
| `GET /replay/{receipt_id}` | हस्ताक्षरित रसीद + `हस्ताक्षर_मान्य`। |
| `POST /verify-receipt` | एक स्टैंडअलोन रसीद को सत्यापित करें (क्रॉस-टूल)। |
| `GET /healthz` | लाइवनेस + कॉन्फ़िगर किए गए सत्यापनकर्ता परिवार (कोई प्रमाणीकरण नहीं)। |

API कुंजियाँ सेट करें (आराम पर हैश की गई) - प्रिज़्म **विफलता-बंद** है, इसलिए `/सत्यापित करें` तब तक अस्वीकार कर दिया जाएगा जब तक कि कुंजियाँ कॉन्फ़िगर नहीं हो जातीं या आप स्थानीय नो-प्रमाणीकरण का विकल्प नहीं चुनते:

```bash
export PRISM_API_KEYS="<sha256(key1)>,<sha256(key2)>"   # callers send: Authorization: Bearer <key>
export PRISM_WEBHOOK_SECRET="<random>"                  # to sign async/escalate webhook deliveries
# local dev only:
export PRISM_HTTP_ALLOW_NO_AUTH=1
```

त्रुटियाँ RFC 9457 `एप्लिकेशन/समस्या+json` हैं; `POST /सत्यापित करें` एक `आइडेंपोटेंसी-कुंजी` हेडर और प्रति-कुंजी दर सीमा (`429` + `पुन: प्रयास-बाद`) का सम्मान करता है। एसिंक्रोनस/एस्केलेट वेबहुक मानक-वेबहुक-हस्ताक्षरित, एसएसआरएफ-संरक्षित (कोई आंतरिक/मेटाडेटा लक्ष्य नहीं), पुन: प्रयास किए जाते हैं, और एक नामित रद्द-घटना क्षतिपूर्तिकर्ता ले जाते हैं।

## रसीदें और हस्ताक्षर (Ed25519, किसी के द्वारा भी सत्यापित किया जा सकता है)

प्रत्येक सत्यापन एक हस्ताक्षरित, दोबारा चलाए जा सकने वाली रसीद `~/.प्रिज़्म/रसीदें.डीबी` में उत्पन्न करता है। v0.4 डिफ़ॉल्ट रूप से नई रसीदों पर **Ed25519 (RFC 8032)** के साथ हस्ताक्षर करता है, इसलिए **एक अलग उपकरण प्रिज़्म की सार्वजनिक कुंजी के साथ एक प्रिज़्म रसीद को सत्यापित कर सकता है - कोई साझा गुप्त कुंजी नहीं**:

```bash
prism keygen --out ~/.prism/signing_key.pem    # generate an Ed25519 keypair
export PRISM_SIGNING_KEY=~/.prism/signing_key.pem
prism pubkey                                    # publish this public key + kid to consumers

# a consumer (e.g. role-os) verifies a receipt with ONLY the public key:
prism verify-receipt receipt.json --public-key prism-pub.pem
```

हस्ताक्षर में निर्णय, पूर्व/पश्च-स्ट्रिप कलाकृति हैश, सत्यापनकर्ता मॉडल, सबमॉड्यूलरिटी मैट्रिक्स, प्रति-लेंस प्रॉम्प्ट हैश (बाइट-दर-बाइट दोबारा चलाए जा सकने योग्य), उद्धरण पुनर्प्राप्ति पिन और हस्ताक्षर `एल्ग`/`किड` शामिल हैं। पुरानी **एचएमएसी** रसीदें अभी भी सत्यापित होती हैं ( `PRISM_SIGNING_SECRET` सेट करें); `PRISM_DEV=1` स्थानीय परीक्षण के लिए एक देव कुंजी बनाता है। प्रिज़्म **सत्यापित/दोबारा चलाएं/सेवा/एमसीपी पथ शुरू करने से इनकार करता है** यदि कोई कुंजी कॉन्फ़िगर नहीं है, बजाय इसके कि सार्वजनिक रूप से ज्ञात कुंजी के साथ चुपचाप हस्ताक्षर किया जाए।

क्षतिपूर्ति आदेशों के साथ संग्रहीत रसीदों का प्रबंधन करें:

```bash
prism receipt delete <receipt_id>
prism receipt prune --older-than 90d --yes
```

## सुरक्षा और गोपनीयता

- **खतरा मॉडल।** प्रिज़्म कलाकृति + आपके द्वारा पारित इरादे और सत्यापनकर्ता मॉडल की प्रतिक्रियाओं को पढ़ता है, और हस्ताक्षरित रसीदों को एक स्थानीय SQLite DB में लिखता है। यह आपके स्रोत ट्री, पर्यावरण या उन प्रदाता API कुंजियों से परे क्रेडेंशियल्स को **नहीं** पढ़ता है जो आप पर्यावरण चर के माध्यम से प्रदान करते हैं। रसीद हस्ताक्षर **तृतीय-पक्ष सत्यापन क्षमता** प्रदान करते हैं (Ed25519: एक उपभोक्ता सार्वजनिक कुंजी के साथ सत्यापित करता है, कोई साझा गुप्त कुंजी नहीं) लेकिन स्थानीय-रूट हमलावर के खिलाफ छेड़छाड़-**प्रूफ** नहीं हैं जो ऑन-डिस्क निजी कुंजी को पढ़ सकते हैं - यह एचएमएसी गुप्त के समान सीमा है। वास्तविक छेड़छाड़-प्रतिरोध के लिए, कुंजी को एक HSM में रखें और रसीदों को एक पारदर्शिता लॉग में एंकर करें (नाममात्र हार्डनिंग पथ)।
- **HTTP सतह।** `प्रिज़्म सेवा` डिफ़ॉल्ट रूप से लूपबैक को बाध्य करता है, **विफलता-बंद** है (API कुंजियों के बिना कोई `/सत्यापित करें` नहीं), आराम पर कुंजियों को हैश करता है, और कॉलर-आपूर्ति किए गए वेबहुक URL को **एसएसआरएफ-सुरक्षित** करता है (कोई आंतरिक/लिंक-स्थानीय/मेटाडेटा लक्ष्य नहीं)। यह कॉलर-आपूर्ति की गई कलाकृतियों को एक मॉडल के माध्यम से चलाता है; एक कलाकृति *प्रॉम्प्ट इंजेक्शन का प्रयास* कर सकती है लेकिन निर्णय योजना या प्रिज़्म की प्रदाता कुंजियों को लीक नहीं कर सकती है।
- **कोई टेलीमेट्री नहीं।** प्रिज़्म केवल उन मॉडल प्रदाताओं को अनुरोध भेजता है जिन्हें आप कॉन्फ़िगर करते हैं (एंथ्रोपिक / ओपनएआई / गूगल / स्थानीय ओलामा)। कुछ भी नहीं।
- पूर्ण नीति: [सुरक्षा.एमडी](सुरक्षा.एमडी)।
